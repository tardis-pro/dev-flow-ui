## Key Design Decisions
- userId pattern: session.user.email (matches keys/route.ts convention)
- CF env access: globalThis pattern (matches bootstrap/route.ts, no runtime change needed)
- Fire-and-forget: IIFE with .catch(console.warn) — response returns before AI finishes
- Informational comment only when cfEnv is available but no key — avoids false negatives in dev
- issueSummary.status = body.toStatus (the NEW status post-move, correct for phase prompt)
